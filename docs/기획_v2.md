## 2. 확장 목표

- PDF는 클라이언트에서 다운로드 (기존 유지)
- 폼 내용(`form_data`)은 Supabase에 저장
- 문서 목록 조회 → 선택 시 폼에 불러오기
- 불러온 문서 수정 후 저장 시 **update**, 신규 작성 시 **insert**
- **템플릿 2종** 지원 (선택·저장·불러오기 시 템플릿 유지)
- 사용자가 PDF/Word 업로드로 템플릿 자동 생성: **범위 외**
- 프로토타입 수준 유지, 과도한 복잡도 지양

## 3. 아키텍처

### 3.1 프론트

- React + Vite 유지 (**Next.js 전환 없음**)
- AWS Amplify Hosting 유지
- `@supabase/supabase-js`로 브라우저에서 직접 API 호출

### 3.2 백엔드 / DB: Supabase

- Postgres + 자동 REST API
- **anon (publishable) key**만 프론트에 사용
- **service_role / DB 비밀번호**는 프론트·`.env`에 넣지 않음
- RLS: 활성화 + anon 전체 허용 (로그인 없음 전제)
- 향후 로그인 도입 시 RLS만 재설계

### 3.3 환경 변수 (Vite)

```env
VITE_SUPABASE_URL=https://xxxx.supabase.co    # /rest/v1/ 붙이지 않음
VITE_SUPABASE_ANON_KEY=sb_publishable_...     # 또는 legacy anon key
```
