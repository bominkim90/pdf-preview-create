## 1. 현황

### 완료된 것

- React + Vite 기반 보고서 작성 앱 (폼 → A4 미리보기 → PDF 다운로드)
- AWS Amplify 정적 배포
- Supabase Auth + `documents` / `profiles` 테이블 + RLS
- 회원가입·로그인 (`@istagingisa.com` 아이디), 마이페이지, 문서 목록
- 비로그인 PDF 작성 (`/guest`)
- OpenAI API 연동 (`.txt` / `.md` → AI 본문 생성)
- `master` 역할: 타인 문서 수정·삭제 가능
- 템플릿 2종 (공문서 보고서, 업무 리스크 관리)

### 환경 변수 (Vite)

```env
VITE_SUPABASE_URL=https://xxxx.supabase.co
VITE_SUPABASE_ANON_KEY=sb_publishable_...
VITE_OPENAI_API_KEY=sk-...
```

- 로컬: `.env`
- 배포: Amplify Environment variables → **재배포 필요**

## 2. DB 스키마 요약

### `profiles`

| 컬럼 | 설명 |
|---|---|
| `id` | `auth.users.id` |
| `email` | 전체 이메일 (`id@istagingisa.com`) |
| `nickname` | 화면 표시용 |
| `role` | `user` (기본) \| `master` |

- 가입 시 `role = user`
- `master` 지정: Supabase SQL Editor에서 수동 `UPDATE`

### `documents`

| 컬럼 | 설명 |
|---|---|
| `author_id` | 작성자 UUID (NOT NULL) |
| `form_data` | 폼 JSON |
| `template_id` | 템플릿 ID |

### RLS

- **authenticated**만 CRUD
- 문서 수정/삭제: 본인 또는 `master`
- 프로필: 본인 insert/update (role 승격 불가 트리거)
